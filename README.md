# GameTerms — ゲーム業界専門用語 学習資料プロジェクト

ゲームプランナー志望者が「企画書 / 企画分析 / 面接 / 制作振り返り / ポートフォリオ」で使える専門用語を、体系的に学習するための資料一式を制作するプロジェクトです。

---

## 1. この資料の目的

- ゲーム業界で使われる専門用語を **意味だけでなく「使う場面」とセットで** 身につける
- まず **用語マスター（一覧）** を整備し、その後で重要度の高い用語から順に **A4両面の学習シート** を作っていく
- 全部を一気に作らず、**10〜20語ずつバッチで追加** していく方式で運用する

主な利用シーン:
- 企画書の中で正しい語彙を使う
- 既存ゲームを分析するときの観点を増やす
- 面接で「自分が何を考えて作ったか」を業界用語で説明できる
- 制作後の振り返りや、ポートフォリオで使う

## 3. 用語マスターのスキーマ

`data/terms_master.json` の `items` 配列に、1用語 = 1エントリで管理する。

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | string | 3桁連番（`"001"` 〜）。決めたら変えない |
| `term` | string | 用語名 |
| `category` | string | 15カテゴリのいずれか（次節） |
| `importance` | string | `"A"` / `"B"` / `"C"` |
| `shortMeaning` | string | 1〜2文の短い意味 |
| `usageScene` | string | どんな時に使うかを **具体的に**（カテゴリ名で済ませない） |
| `sheetCreated` | boolean | 学習シート作成済みかどうか。初期は `false` |
| `notes` | string | 補足。なければ空文字 |

### カテゴリ一覧（15）
ゲーム企画 / ゲームデザイン / レベルデザイン / UI / UX / プレイヤー体験 / ゲームシステム / 開発工程 / プロトタイプ・検証 / テスト・改善 / 運営・KPI / マネタイズ / マーケティング / ポートフォリオ・面接 / Unity / 開発実装 / サウンド・演出

### 重要度
- **A**：ゲームプランナー志望なら最優先で覚える
- **B**：知っておくと企画・制作で役立つ
- **C**：必要になった時に確認すればよい

## 4. 用語を追加する方法

### 4-1. 用語マスター（JSON）に追記
1. `data/terms_master.json` を開く
2. `items` 配列の末尾にエントリを追加する（既存の最大IDの次番号）
3. 同じ用語が既にないか必ずチェック
4. `shortMeaning` は1〜2文、`usageScene` は **カテゴリ名で済ませず具体的に書く**
5. `sheetCreated` は `false` で登録

### 4-2. CSVも更新（任意）
`data/terms_master.csv` は閲覧・編集の補助用。JSON が正本なので、CSVは「JSONを直したらコピペで合わせる」運用でよい。

### 4-3. terms_master.html のデータも更新
`terms_master.html` は内部に同じ配列を埋め込んで動作している。
用語追加時は HTML 内の `const TERMS = [ ... ]` にも同じエントリを追加する。

> 将来的には JSON を fetch する形に切り替えてもよい。当面は3か所（JSON / CSV / HTML内）を同期させる前提。

## 5. `sheetCreated` の使い方

`sheetCreated` は、その用語の **学習シートを作ったかどうか** を表すフラグ。

| 値 | 意味 | 一覧ページでの見え方 |
|---|---|---|
| `false` | まだ学習シートを作っていない | `—` |
| `true` | 学習シート作成済み | `✓`（緑） |

### 運用ルール
- 新しく用語を追加した時は **必ず `false`** から始める
- 学習シートを作って `output/html/<id>.html` に保存したら、その用語の `sheetCreated` を `true` に更新する
- `terms_master.html` の「シート」フィルタで「作成済 / 未作成」を切り替えると、進捗が一目でわかる
- 「次にどの用語をシート化するか」は **重要度A かつ `sheetCreated: false`** から選ぶのが基本

## 6. 次に学習シートを作る時の流れ

学習シート化は、いきなり大量に作らず **10〜20語ずつのバッチ** で進める。

```
【1バッチ = 10〜20語】の流れ

  ① 対象の用語を選ぶ
       重要度A の中から sheetCreated:false のものを優先
       ↓
  ② 1用語ずつ学習シート用の中身を埋める
       ・詳しい解説  ・背景/由来
       ・使われる場面/職種   ・例文（2つ）
       ・関連用語   ・確認問題（2問）
       ↓
  ③ templates/term_sheet_template.html のプレースホルダに流し込む
       → output/html/<id>_<term>.html を生成
       ↓
  ④ ブラウザで印刷プレビュー → A4両面で2ページに収まるか確認
       必要なら output/pdf/ に PDF 出力
       ↓
  ⑤ data/terms_master.json の対象用語を sheetCreated:true に更新
       terms_master.html 内の TERMS 配列も同期
       ↓
  ⑥ data/batches/batch_NNN.json に作成記録を残す
       （対象ID一覧、作成日、メモ）
```

### バッチ記録のフォーマット例
`data/batches/batch_001.json`:
```json
{
  "batchId": "batch_001",
  "createdAt": "2026-05-08",
  "targetIds": ["001","002","003","008","009","011","033","049","071","091"],
  "memo": "重要度A の最初の10語。企画コア / 体験コア / 仕様書 / KPI / ポートフォリオを優先。"
}
```

---

## 7. 進め方の原則

- **品質優先・量は後**：意味の薄い用語を増やすより、本当に使う用語を厚く
- **段階的に拡張**：マスター → 一覧UI → A の学習シート → B → C の順で
- **重複に注意**：似た意味の用語は notes で関係性を書く（例:「ターゲットユーザーが層、ペルソナが個人」）
- **使う場面を必ず書く**：「企画書・面接で使う」のような曖昧な書き方はしない
